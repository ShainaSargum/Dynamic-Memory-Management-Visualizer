Segmentation - basics

Memory ko variable size logical blocks (segments) me divide karte hain.

BASE - physical starting address
LIMIT - max size (offset is checked against this)
OFFSET - distance inside the segment
Valid if offset < limit, otherwise segmentation fault.

Physical address = base + offset

Virtual Memory

Virtual memory ek technique hai jisme program ko aisa lagta hai ki uske paas ek bahut bada continuous memory hai, jabki actually OS RAM + disk (swap) dono ka use karta hai. CPU jo virtual address generate karta hai use MMU physical address me translate karta hai.

Need / kyon use karte hain

- Programs RAM se bade bhi run ho sakte hain
- Har process ka apna isolated address space hota hai (security)
- Memory sharing easy ho jata hai
- Linker / loader ka kaam simple ho jata hai

Important terms

Virtual address - CPU jo address generate karta hai
Physical address - actual RAM ka location
Page table - virtual page -> physical frame mapping
MMU - hardware unit jo translate karta hai
TLB - cache for recent page table entries (fast lookup)
Swap space - disk pe wo area jaha pages temporarily store hote hain

Demand paging

Page tabhi RAM me lao jab actually use ho. Agar page nahi mila to page fault aata hai, OS disk se page laata hai aur (zarurat hone par) FIFO / LRU / OPTIMAL se kisi purane page ko replace karta hai.

Translation flow

1. CPU virtual address banata hai = page number + offset
2. MMU pehle TLB check karta hai, miss ho to page table dekhta hai
3. Agar page RAM me hai - physical address = frame * pageSize + offset
4. Nahi to page fault -> OS disk se laata hai -> instruction retry hoti hai

Thrashing

Jab system useful kaam se zyaada time pages swap karne me lagata hai. Reason: bahut zyada processes, kam frames. Solution - working set model, load control.

Effective Access Time (EAT)

EAT = (1 - p) * memory_access + p * page_fault_time
Jaha p = page fault rate. Choti si p value bhi performance kharab kar sakti hai kyunki disk access RAM se bahut slow hota hai.

Pros / Cons

Pros: bada logical address space, isolation, multiprogramming, on-demand loading.
Cons: page fault overhead, thrashing risk, hardware complexity.
